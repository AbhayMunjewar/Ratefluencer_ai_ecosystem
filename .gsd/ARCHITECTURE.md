## Components

The application divides components into two main categories: shared layout/global UI components and pages that contain feature-specific layouts and components.

### Layout & Global Components (`src/app/components/`)
- **`Sidebar.tsx`**: Left-side navigation bar with responsive active link styling.
- **`Topbar.tsx`**: Header containing global search, system alerts, notifications, and user profile state.
- **`GlassCard.tsx`**: Shared styled card component implementing glassmorphic borders and blur backdrops.
- **`KPICard.tsx`**: Reusable card displaying key performance indicators (KPIs) with colored sparklines and trends.
- **`AIScoreGauge.tsx`**: Reusable gauge component demonstrating authenticity and risk scoring using circular indicators.
- **`AtmosphericBackground.tsx`**: Shared visual styling element for background glows and premium gradients.
- **`SparkLine.tsx`**: Renders micro SVG sparklines for compact historical trend visualisation.

### Feature Pages (`src/app/pages/`)
- **`Landing.tsx`**: Marketing landing page detailing the ecosystem values and capabilities, heavily animated.
- **`Dashboard.tsx`**: Core operations overview displaying KPI cards, recent feeds, and analytics graphs.
- **`InfluencerSearch.tsx`**: Multi-dimensional search grid to filter, rank, and view top influencers.
- **`InfluencerProfile.tsx`**: Deep-dive profile page for a single influencer showing historical data, platform statistics, and brand matches.
- **`AuthenticityAnalysis.tsx`**: Specialized page to analyze and flag fake follower counts, engagement manipulation, and high-risk behaviors.
- **`GrowthPrediction.tsx`**: Renders AI-driven growth trajectory projections over 6-month time horizons.
- **`BrandMatching.tsx`**: Calculates and reports compatibility matching scores between brand goals and influencer attributes.
- **`Campaigns.tsx`**: Campaign manager allowing marketers to track progress, deliverables, budget allocations, and ROI.
- **`Leaderboard.tsx`**: Renders rankings of top performers mapped across distinct industry categories.
- **`AICopilot.tsx`**: Interactive chat assistant utilizing mock AI logic to help marketers answer platform questions.
- **`AdminPanel.tsx`**: Administrative system controls and configuration dashboard.
- **`Settings.tsx`**: User-specific display, notification, and profile management settings.

## Data Flow

Data in the current front-end is managed using mock data sets to simulate live API endpoints:
1. **Mock Data Injection**: Located in `src/data/`, containing static JSON files for influencers, campaigns, activity feeds, and trends.
2. **State Management**: Component local state (`useState`) handles interactive inputs, filter configurations, search term updates, and chat history.
3. **Lazy Loading**: Code splitting is configured in `routes.tsx` using `lazy` and `Suspense` with an animated `PageLoader` component to manage route transitions.

## Technical Debt

- **Missing Backend Integration**: The system relies purely on mock data JSONs and client-side simulation.
- **Static Assets**: Several dashboard elements use mock icons or placeholder indicators instead of dynamic media files.
- **No Shared Global State Store**: Routing inputs are passed around or read from local mock imports directly. A shared store (e.g. Zustand or Context) will be required when scaling API requests.
- **Lack of Automated Testing**: There are no unit or end-to-end tests set up within the `frontend/` directory.

## Conventions

- **Naming**: UpperCamelCase for React components and page modules (`Dashboard.tsx`); camelCase for utilities, hooks, and JSON data.
- **Structure**: Separation of layouts, components, page routes, static data stores, and design system styles.
- **Styling**: Vanilla CSS custom variables combined with Tailwind v4 `@tailwindcss/vite` imports.
